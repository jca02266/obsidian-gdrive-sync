# Obsidian Google Drive Sync (Fork版) - 日本語ガイド

このドキュメントでは、本プラグイン（[stravo1/obsidian-gdrive-sync](https://github.com/stravo1/obsidian-gdrive-sync) からのフォーク版）におけるオリジナルのプラグインからの主な新機能、バグ修正、および改善点について解説しています。日本のユーザーにとっての最初の参照ページとしてご活用ください。
## 🎨 UI・通知の改善

- **同期通知の静音化 (UXの向上)**  
  バックグラウンドのファイル同期（アップロード・ダウンロード）のたびに表示されていた画面中央のポップアップ通知（Toast）を大幅に削減しました。代わりに、画面右下のステータスバーアイコンと、マウスオーバー時のツールチップ（`aria-label`）に進捗状況を表示するようになり、作業の妨げになりません。
- **ダウンロード進捗表示の適正化**  
  大量のファイルをダウンロードする際、途中で再開したときに「最初からやり直している」と誤解を招きやすい進捗表示を、常に「現在の進捗数 / 全体のファイル数」ベースの表示に変更しました。
- **「Vault接続リセット」ボタンの追加**  
  設定画面から、Google DriveのVault接続情報（`vaultInit`, `vaultId`, `filesList`など）を安全にリセットできるボタンを追加しました。これにより、プラグインのデータファイル（`data.json`）を直接編集する手間が省け、バックエンドの移行や再設定が容易になりました。

## 🚀 同期パフォーマンスと安定性の向上

- **APIコールの最適化（高速化）**  
  ファイルのダウンロード時、すでにファイル名が判明している場合はGoogle Drive APIへのファイル名取得リクエストを省略するよう `getFile` 関数を最適化しました。これにより、大量のファイル同期時のAPIコール数が半減し、ダウンロード速度が約2倍に向上しました。
- **ダウンロード処理の堅牢化**  
  同期ループ中に1つのファイルで通信エラーなどが発生した場合でも、処理全体を停止させず、そのファイルだけをスキップして他のファイルのダウンロードを続行するようにエラーハンドリングを強化しました。
- **フォルダ検索のルート制約**  
  Google Drive上でプラグインのフォルダ（`obsidian` 等）を検索する際、マイドライブのルート階層（`root`）のみを対象とするように修正しました。これにより、他の階層に同名の過去フォルダが存在することによる重複エラーや不具合を防いでいます。

## 🐛 エラーハンドリング・ログの改善

- **ネットワークエラー時のログ軽減**  
  DNS失敗など一時的なネットワークエラーが発生した場合、エラーログ（`error-log-gdrive-plugin.md`）に長大なスタックトレースを出力せず、エラー名とメッセージのみを簡潔に記録するよう変更し、ログの肥大化を防ぎました。
- **Vaultフォルダ削除時の対応**  
  Google Drive側で直接Vaultフォルダが削除され 404 エラーが返された場合、通信再試行のループに陥るのを防ぎ、自動で `vaultId` などの設定をリセットしてユーザーに再初期化を促すよう修正しました。

## 📄 ドキュメントとビルド環境の整備

- **独自のOAuthバックエンドサーバーの実装とドキュメント化**  
  フルアクセススコープを利用するための独自のバックエンドサーバー実装（`backend/server.js`）と、Google Cloud RunやRender等のPaaSへのデプロイガイド（`backend/README.md`, `backend/README.render.md`）を追加しました。
- **Render手順の最適化**  
  実際のRenderのUIやObsidianプラグインの設定画面に合わせた手順へとドキュメントを更新しました。
- **ビルドスクリプトの改善**  
  `npm run build` を実行した際、リリースや手動インストール用の `obsidian-gdrive-sync.zip` が自動で生成されるようになり、リリースの手間が軽減されました。
